## Why this matters

A 4.2 MB Markdown dump of old test output sits in the repo root, and the
frontend ships an entire drag-and-drop library (`@dnd-kit/*`) that nothing
imports (the app uses `@hello-pangea/dnd`). Both are pure clutter: the big file
bloats clones and confuses anyone trying to read current test status; the unused
dep adds install weight and supply-chain surface for zero benefit. Removing them
is safe and quick.

- `test_failures_2025-11-19.md` — 4.2 MB file in repo root (verbose Jest/Playwright
  error traces dated 2025-11-19). Not referenced by any code (verify in Step 1).

  ```json
      "@dnd-kit/core": "^6.1.0",
  ```
